# SRE-STUDY 系统安全与性能审计自动化
## 项目简介
  本项目暂时主要基于Python3 开发的SRE自动化监控方案。目前主要实现了底层日志流实时监听、内存和磁盘指标采集、安全风险自动化分析。
  目前项目状态:【Developping】

## 核心架构
  该项目主要采用事件驱动 (Event-Driven) 与 流式处理 (Streaming)，主要包含以下核心功能: 
1. 实时日志审计
    * 流式监听: 模拟```tail -f```,控制文件指针偏移，实现对日志的增量读取
    * 正则结构化: 利用正则表达式命名组 (?P\<name>) ,将SSH登录日志解析为结构化数据

2. 多维指标监控
    * 内存画像: 实时采集系统可用内存、使用率及状态
    * 磁盘审计: 支持指定挂载点的空间分析，包括总量、已用、剩余及百分比监控。
    * 硬件预警: 内置阈值判定逻辑，自动识别 HEALTHY、WARNING 或 DANGER 状态。

3. 安全分析引擎
    * 攻击画像: 利用 collections.Counter 实时更新高频登录错误 IP 账本
    * 风险评估: 内置智能评估逻辑，根据频率自动判定风险等级（NORMAL / WARNING / CRITICAL）
    * 防抖机制: 实现时间窗口控制，防止在高频事件触发下造成的系统 IO 过载

4. 自动化报告
    * 动态报表: 可自动生成 Markdown 格式的系统审计报告
    * 可视化渲染: 将复杂数据转化为 ASCII 柱状图及状态指示器

# 学习方向根据与依赖
- **工程化思维**: 坚信任何重复性、手动的工作都可以且应该通过代码（自动化）来完成。
- **量化思维**: 习惯用数据说话。引入服务等级目标（SLO）和错误预算来精确衡量系统稳定性。

## 🗺️ SRE 学习路线图

### 第一阶段

**目标**: 掌握 Linux 系统管理和网络基础，为后续一切技术打下根基。

| 类别           | 核心内容                                                     |
| :------------- | :----------------------------------------------------------- |
| **Linux 系统** | 命令行操作、文件/用户/进程/权限管理、系统日志（systemd/journald）、软件包管理。 |
| **网络基础**   | OSI 模型、TCP/IP 协议栈（DNS/HTTP/HTTPS）、子网划分、VLAN、路由基础。 |
| **编程入门**   | 开始学习 **Python** 或 **Go**，为自动化编写脚本打下基础。    |

---

### 第二阶段

**目标**: 拥抱云原生，掌握现代化的应用部署与运维方式。

| 类别                     | 核心内容                                                     |
| :----------------------- | :----------------------------------------------------------- |
| **容器化与编排**         | 深入学习 **Docker**（构建、镜像优化）和 **Kubernetes**（Pod、Deployment、Service、Ingress）。 |
| **可观测性**             | 学习使用 **Prometheus** 采集指标、**Grafana** 可视化，搭建 **ELK/Opensearch** 日志系统。 |
| **基础设施即代码 (IaC)** | 学习使用 **Terraform** 或 **Ansible**，通过代码管理基础设施。 |

---

### 第三阶段

**目标**: 从保障系统“可用”升级到保障系统“弹性”，具备架构设计与全局视角。

| 类别               | 核心内容                                                     |
| :----------------- | :----------------------------------------------------------- |
| **性能与压力测试** | 学习使用 **JMeter**、**k6** 进行压测，定位性能瓶颈。         |
| **混沌工程**       | 主动注入故障（如随机 Kill Pod），验证系统自愈和容错能力。    |
| **深入分布式系统** | CAP 理论、一致性协议、微服务架构、消息队列（Kafka）、系统安全。 |
| **成本与容量管理** | 精确的容量规划，在保证可靠性的前提下控制云资源成本。         |
